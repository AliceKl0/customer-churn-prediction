# Результаты анализа оттока клиентов

В этой папке собраны все визуализации и графики, полученные в ходе анализа данных и обучения моделей машинного обучения.

## Описание файлов

### 1. `eda_visualizations.png`
**Разведочный анализ данных (EDA)**

Содержит 6 графиков, показывающих основные закономерности в данных:
- **Распределение оттока**: показывает соотношение клиентов, которые ушли и остались
- **Возраст и отток**: boxplot, демонстрирующий связь между возрастом клиента и вероятностью оттока
- **Кредитный рейтинг и отток**: визуализация зависимости оттока от кредитного рейтинга
- **Баланс счета и отток**: анализ влияния размера баланса на отток
- **Отток по странам**: процент оттока в разных географических регионах
- **Отток по активности**: сравнение оттока между активными и неактивными клиентами

**Выводы:**
- Клиенты старшего возраста (60+) чаще уходят из банка
- Низкий кредитный рейтинг (<500) коррелирует с повышенным оттоком
- Неактивные клиенты уходят чаще активных
- Географические различия могут указывать на региональные особенности обслуживания

### 2. `correlation_matrix.png`
**Корреляционная матрица признаков**

Тепловая карта показывает корреляции между всеми числовыми признаками в датасете. Значения от -1 (отрицательная корреляция) до +1 (положительная корреляция).

**Выводы:**
- Сильные корреляции между признаками могут указывать на мультиколлинеарность
- Признаки с высокой корреляцией с целевой переменной (exited) наиболее важны для предсказания
- Низкие корреляции между большинством признаков - хороший знак для моделей

### 3. `model_performance.png`
**Сравнение производительности моделей**

Матрицы ошибок (confusion matrices) для всех обученных моделей. Показывает:
- **True Negative (TN)**: правильно предсказанные "остались"
- **False Positive (FP)**: неправильно предсказанные "ушли" (ошибка I типа)
- **False Negative (FN)**: неправильно предсказанные "остались" (ошибка II типа)
- **True Positive (TP)**: правильно предсказанные "ушли"

**Выводы:**
- Модель с наименьшим количеством ошибок (особенно FN) наиболее полезна для бизнеса
- False Negative особенно критичны - это клиенты, которые уйдут, но мы их не выявили
- ROC-AUC на графике показывает общее качество модели (чем выше, тем лучше)

### 4. `roc_curves.png`
**ROC-кривые для сравнения моделей**

ROC-кривые показывают соотношение между True Positive Rate (TPR) и False Positive Rate (FPR) при разных порогах классификации.

**Выводы:**
- Кривая, расположенная выше и левее, соответствует лучшей модели
- Площадь под кривой (AUC) показывает способность модели различать классы
- AUC > 0.8 считается хорошим результатом
- Модель с AUC близким к 1.0 практически идеально разделяет классы
- Случайный классификатор (пунктирная линия) имеет AUC = 0.5

### 5. `feature_importance.png`
**Важность признаков для tree-based моделей**

Горизонтальный бар-график показывает топ-15 наиболее важных признаков для Random Forest или XGBoost модели.

**Выводы:**
- Признаки вверху списка наиболее критичны для предсказания оттока
- Эти признаки должны быть приоритетными при сборе данных и мониторинге
- Низкая важность признака не означает, что его можно удалить - он может работать в комбинации с другими
- Созданные в feature engineering признаки (например, balance_salary_ratio) часто оказываются важными

## Общие выводы и рекомендации

1. **Качество моделей**: Все модели показывают приемлемое качество (ROC-AUC обычно > 0.75). Лучшая модель после тюнинга гиперпараметров должна использоваться в продакшене.

2. **Ключевые факторы оттока**:
   - Возраст клиента (старше 60 лет)
   - Кредитный рейтинг (низкий)
   - Активность клиента (неактивные уходят чаще)
   - Финансовые показатели (баланс, соотношение баланса и зарплаты)

3. **Рекомендации для бизнеса**:
   - Фокус на клиентах старшего возраста - разработать специальные программы удержания
   - Мониторинг клиентов с низким кредитным рейтингом
   - Повышение активности клиентов через персонализированные предложения
   - Использование модели для проактивного выявления клиентов группы риска

4. **Дальнейшие шаги**:
   - Развертывание лучшей модели в продакшене
   - Настройка порога классификации в зависимости от бизнес-целей (минимизация FN или FP)
   - Регулярное переобучение модели на новых данных
   - A/B тестирование различных стратегий удержания клиентов
